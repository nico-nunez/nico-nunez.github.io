Standalone Groovebox for macOS

github.com/nico-nunez
  • Standalone C/C++ performance instrument combining a shared clock, pattern-based step sequencer, and multi-track synthesis — built from scratch without audio framework abstractions

  • Embedded a Lua runtime as a live REPL control surface, enabling real-time parameter changes, sequencer control, and performance scripting without recompiling

  • Hosts the synth engine library; built-in code editor with language server support
C/C++, CoreAudio, CoreMIDI, Lua, Make, macOS

Embeddable C/C++ Synthesizer Library

github.com/nico-nunez
  • Standalone C/C++ library extracted from the groovebox application — builds as a static lib and can be embedded into other projects via engine.mk; designed with a clean host/engine boundary so the engine owns synthesis state and the host owns platform concerns
  • Implements 4 wavetable oscillators per voice, 64-voice polyphony, 3 envelope generators, 3 LFOs with tempo-sync and fade-in, and a 16-slot modulation matrix with sources including envelopes, LFOs, velocity, mod wheel, and key tracking
  • Renders in internal 64-sample chunks regardless of host buffer size, providing a cheap control-rate lane; parameters use a dirty-sync model to defer coefficient recalculation rather than recomputing on every write
  • Preset system captures full engine state — scalar params, FM routing, modulation matrix, signal and FX chain ordering — with JSON serialization for persistence
  • Event-driven API surfaces three input channels: MIDI events, scalar parameter updates, and higher-level engine commands
C++17, Make, static library, JSON
